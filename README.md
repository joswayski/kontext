# Kontext

⚠️ This project is in the very early stages of development ⚠️

## Overview

I am looking to build a tool that will help me:
- Understand the flow of events within my Kafka cluster(s)
- Track event schema changes
- Figure out what my services *do* with the data (business logic)


## Running

```bash
cp .env.example .env
```

```bash
docker compose up -d
```

Then, the Web app will be available at `http://localhost:3000` and the API will be available at `http://localhost:4000`.
